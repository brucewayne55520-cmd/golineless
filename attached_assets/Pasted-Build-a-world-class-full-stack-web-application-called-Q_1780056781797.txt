GPS TRACKING SYSTEM (Core Feature)
════════════════════════════════
Use Leaflet.js + OpenStreetMap tiles.

RUNNER SIDE — GPS broadcast:
- When runner accepts task, start broadcasting location
- Use browser Geolocation API: navigator.geolocation.watchPosition()
- Emit via Socket.io: socket.emit('runner_location', {taskId, lat, lng, heading})
- Update every 3 seconds
- Show runner current speed and ETA calculation

USER SIDE — Live map view:
- Full-screen map in task detail page (like Zomato order tracking)
- Animated runner marker (use custom purple pin icon)
- Pulsing circle around runner marker (like Blinkit)
- User location marker (home icon)
- Destination marker (flag icon)
- Polyline route between runner → destination (draw straight line,
update as runner moves)
- Animated dotted path trail showing where runner came from
- ETA display: "Runner arrives in ~12 mins" (calculated from distance)
- Live status bar at bottom of map:
"Mohammed is on the way 🏃" with runner photo + name
- Map auto-pans to keep runner centered as they move
- Zoom controls
- "Runner reached!" animation (confetti burst) when runner marks arrived

BACKEND — Location storage:
CREATE TABLE runner_locations (
runner_id INT,
task_id INT,
lat DECIMAL(10,8),
lng DECIMAL(11,8),
heading DECIMAL(5,2),
speed DECIMAL(5,2),
recorded_at TIMESTAMP DEFAULT NOW()
);

Socket.io rooms: each task gets room `task_${taskId}`
Runner joins room on task accept
User joins room on task detail page open
Admin can see all active runners on admin map

DATABASE SCHEMA (complete)
════════════════════════════════

All tables from before PLUS:

CREATE TABLE notifications (
id SERIAL PRIMARY KEY,
user_id INT,
runner_id INT,
type VARCHAR(50),
title VARCHAR(200),
message TEXT,
is_read BOOLEAN DEFAULT false,
task_id INT,
created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE runner_locations (
id SERIAL PRIMARY KEY,
runner_id INT,
task_id INT,
lat DECIMAL(10,8),
lng DECIMAL(11,8),
heading DECIMAL(5,2) DEFAULT 0,
speed DECIMAL(5,2) DEFAULT 0,
recorded_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE reviews (
id SERIAL PRIMARY KEY,
task_id INT,
user_id INT,
runner_id INT,
rating INT CHECK (rating BETWEEN 1 AND 5),
review TEXT,
created_at TIMESTAMP DEFAULT NOW()
);
